## 2026-06-23 - [Prevent DoS via Payload Limits and Add Security Headers]
**Vulnerability:** The proxy's `collect(stream)` function accumulated incoming stream data unbounded. This could lead to a Memory Exhaustion Denial of Service (DoS) attack if a malicious user sent a massive payload. Furthermore, the `corsHeaders()` function returned very permissive headers and lacked basic defense-in-depth security headers like `Strict-Transport-Security`, `X-Content-Type-Options`, and `X-Frame-Options`.
**Learning:** For a local proxy handling streaming data, it's easy to assume local traffic is safe, but unbounded array pushes without a size check leave the proxy vulnerable. Rejections from streams in async code must also be properly caught to avoid crashing the server.
**Prevention:** Always enforce a payload size limit (e.g., 10MB) in streams using `stream.destroy()` if exceeded. Wrap promise await calls in `try/catch` and gracefully handle error types (e.g., return HTTP 413 Payload Too Large). Always inject standard HTTP security headers (HSTS, nosniff, DENY frame options).
