## 2026-06-20 - Config File Sync Read Bottleneck
**Learning:** The `loadConfig` function inside the local proxy (`proxy-cors.js`) previously executed `fs.readFileSync` and `JSON.parse` synchronously on *every single incoming request* to enable hot-reloading. This design blocks the Node.js event loop proportionally to the number of requests, causing measurable latency spikes under concurrent request load (e.g., when the Word Add-in sends validation pings or streaming inputs).
**Action:** Always check high-frequency functions (like route handlers) for synchronous I/O. Use a time-based or `fs.statSync`-based debounce cache (e.g., check `mtimeMs` at most every 2 seconds) instead of blindly reading from disk, preserving hot-reload functionality without blocking the event loop constantly.
